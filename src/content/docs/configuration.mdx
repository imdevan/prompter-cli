Configuration file location: `~/.config/prompter/config.toml`

## Basic Config

```toml  title="config.toml"
# Template directory
prompts_location = "~/.config/prompter/prompts"

# Local templates directory name (in current working directory)
local_prompts_location = "prompts"

# Editor for opening templates
editor = "code"

# Output destination: "clipboard", "stdout", or "file:/path"
target = "clipboard"

# Default to interactive mode
interactive_default = true

# Directory listing strategy: "git" or "filesystem"
directory_strategy = "git"

# Default templates (optional)
default_pre = ""
default_post = ""

# Fix mode output file
fix_file = "/tmp/prompter-fix.txt"
```

## Custom Template Locations

Add custom template directories with their own CLI flags:

```toml title="config.toml"
[custom_template.work]
location = "~/work-templates"
flag = "work"
shorthand = "w"
type = "pre"
interactive = true
description = "work-specific templates"

[custom_template.review]
location = "~/review-templates"
flag = "review"
shorthand = "r"
type = "post"
description = "code review checklists"
```

This creates `--work` / `-w` and `--review` / `-r` flags.

### Custom Template Options

| Option | Default | Description |
|--------|---------|-------------|
| `location` | `prompts_location/name` | Template directory path |
| `flag` | template name | CLI flag name |
| `shorthand` | none | Single character shorthand |
| `type` | `"pre"` | `"pre"` or `"post"` |
| `interactive` | `true` | Show in interactive mode |
| `description` | auto | Help text |

## Environment Variables

Override config with `PROMPTER_` prefix:

```bash
export PROMPTER_EDITOR="nvim"
export PROMPTER_TARGET="stdout"
export PROMPTER_INTERACTIVE_DEFAULT="false"
```

## Config Precedence

1. Command-line flags (highest)
2. Environment variables
3. Config file
4. Built-in defaults (lowest)

## Custom Config Path

```bash
prompter --config /path/to/config.toml list
```
